# План абляционных экспериментов (Phase 4B)

Цель: изолированно проверить влияние каждого изменения на f1_macro и достичь прироста минимум +0.33% (абс.) относительно текущего бейзлайна.

- Primary metric: `f1_macro` (валидация)
- Secondary: `balanced_accuracy`, `loss`, `accuracy`, `precision_macro`, `recall_macro`
- Протокол: фиксированный seed (=42), одинаковые сплиты, один фактор меняется за раз
- Оценка: усреднение по 5 фолдам (или по Fold 1 для быстрых прогонов); при наличии 5 фолдов — парный t-test на дельту F1
- Порог значимости улучшения: +0.33% (абс.)

## Наборы данных
- DS1 (baseline): `sub-04`, Fold 1 (как в текущем репо)
- DS2 (multi-subject): `sub-01..sub-04`, без агрессивного undersampling (использовать все доступные данные)
- DS3 (subject-aware CV): `StratifiedGroupKFold` (group=subject) или LOSO

Для изолированных абляций начните с DS1. После подтверждения эффекта — реплицируйте на DS2/DS3.

## Фиксированные параметры (если эксперимент явно не меняет)
- `n_epochs=50`, `batch_size` как по умолчанию, `optimizer=AdamW`, `weight_decay=1e-4`
- `scheduler=cosine`, `T_max=50`, `warmup_epochs=5` (рекомендуемая настройка для всех абляций по умолчанию)
- `normalize='zscore_hybrid'`, `exclude_channels=[124]`, `cov_type='corr'`

Для каждой абляции:

## A1 — WeightedRandomSampler (баланс в батчах)
- Гипотеза: Балансировка градиентов по классам без потери данных повышает f1_macro.
- Изменение (train loader):
  - Вычислить веса классов: `w_c = total / (num_classes * count_c)`
  - Вес сэмпла i: `weights[i] = w_c[label_i]`
  - `WeightedRandomSampler(weights, num_samples=len(train_idx), replacement=True)`
  - DataLoader(train, sampler=sampler, shuffle=False)
- Датасет: DS1
- Критерий успеха: +0.33%
- Риски: шумные миноритарные примеры могут переусилиться (компенсируется CB-Focal).

### Результаты (DS1, Fold 1)
- Early stopping: эпоха 21 (лучшая модель)
- Метрики:
  - accuracy: 0.2474
  - f1_macro: 0.2266
  - precision_macro: 0.3076
  - recall_macro: 0.2876
  - balanced_accuracy: 0.2876
  - loss: 1.4887
- Сравнение с бейзлайном (sub-04, Fold 1):
  - f1_macro: 0.2737 → 0.2266 (−0.0471)
  - accuracy: 0.2874 → 0.2474 (−0.0400)
  - balanced_accuracy: 0.2929 → 0.2876 (−0.0053)
  - loss: 1.4607 → 1.4887 (+0.0280)
- Вывод: чистое включение WeightedRandomSampler при активном Class-Balanced Focal Loss ухудшило макро-F1; прирост цели (+0.33%) не достигнут.
- Гипотезы причин:
  - Двойная балансировка (sampler + CB-Focal) переусиливает редкие/сложные примеры, увеличивая дисперсию градиентов и шум в обучении.
  - Семплирование с заменой вызывает частые повторения миноритарных сэмплов в эпохе → снижается покрытие разнообразия.
- Рекомендации:
  - R1: Отключить sampler при CB-Focal и перейти к A2 (подбор `gamma`/`beta`).
  - R2: Смягчить sampler: `weights_per_class **= alpha`, где `alpha∈[0.5,0.8]`; или смешивание с uniform: `w = λ·w + (1−λ)` с `λ∈[0.5,0.8]`.
  - R3: Если оставлять sampler — заменить CB-Focal на CrossEntropy (или снизить `gamma≈1.25`).
- Решение: перейти к A2; подготовить ветку A1' с soft-sampler (`alpha=0.5`) для повторной проверки.

## A6 — Subject embeddings: размерность и регуляция
- Гипотеза: Увеличение размерности до 32/64 + лёгкая регуляция повышают переносимость.
- Изменение: `subject_embed_dim ∈ {16, 32, 64}`; добавить Dropout(0.1–0.2) на эмбеддинг (если потребуется) или L2.
- Датасет: DS1 → DS2
- Критерий: +0.33%
- Риски: переобучение при большой размерности без регуляции.

### Результаты (DS1, Fold 1) — свип dim × dropout × embed_wd
- Лучший по f1_macro: `dim=16`, `dropout=0.2`, `embed_wd=5e-4`
  - accuracy: 0.3090
  - f1_macro: 0.2915
  - precision_macro: 0.3066
  - recall_macro: 0.3055
  - balanced_accuracy: 0.3055
  - loss: 1.4058
- Второй сильный (высокий acc, умеренный f1): `dim=64`, `dropout=0.2`, `embed_wd=1e-3`
  - accuracy: 0.3123
  - f1_macro: 0.2893
  - precision_macro: 0.2945
  - recall_macro: 0.3064
  - balanced_accuracy: 0.3064
  - loss: 1.4054
- База (A2, dim=16, drop=0.0, wd=same): f1_macro 0.2908, acc 0.3026, loss 1.4384

### Анализ
- Размерность:
  - 16D с регуляризацией (drop=0.2, wd=5e-4) даёт наилучший f1_macro; 32D чаще деградирует f1; 64D повышает accuracy/recall, но f1 уступает 16D‑лучшим.
- Регуляризация:
  - Dropout на эмбеддинге полезен (0.2 > 0.1 > 0.0 на 16D); мягкий L2 на эмбеддинг (5e-4) помогает на 16D; на 64D лучше 1e-3.
- Итог против A2:
  - Прирост f1_macro невелик: 0.2915 vs 0.2908 (+0.0007, +0.07% абс.) — статистически сомнительно на одном фолде; требуется 5-fold проверка.

### Рекомендации
- Кандидат на дефолт для DS1: `dim=16`, `dropout=0.2`, `embed_wd=5e-4` (условно до 5-fold подтверждения).
- Провести 5-fold DS1 и DS2 для проверки устойчивости выбранной тройки параметров.
- Если цель — выше accuracy/recall при минимальной потере f1, рассмотреть `dim=64`, `dropout=0.2`, `embed_wd=1e-3` (но f1 ниже 16D‑лучшего).

- std=0.03, prob=0.3 → acc=0.2946, f1=0.2698, prec=0.2851, rec=0.2872, bal_acc=0.2872, loss=1.4008

### Анализ
- Тренд по вероятности: `prob=0.2` > `0.3` по f1_macro во всех парах std.
- Тренд по амплитуде: при `prob=0.2` умеренный шум (`std=0.03`) > малый (`0.02`) по f1 и loss.
- Сравнение с бейзлайнами: против README-бейзлайна (f1=0.2737) — +0.24 п.п. (не достигает порога +0.33 п.п.); против A2 best (f1=0.2908) — ниже.
- Побочный эффект: loss снижается у лучших настроек (1.3897), accuracy растёт, но макро-F1 не превосходит A2.

### Рекомендации
- Не включать по умолчанию; оставить как опцию. Подтвердить лучший режим (`std=0.03`, `prob=0.2`) на 5-fold DS1 и проверить DS2.
- При дальнейшем поиске рассмотреть более консервативные `std∈{0.01,0.015}` и/или уменьшение `prob≤0.2`, а также аннейлинг вероятности по эпохам.

## A9 — EMA параметров (Polyak averaging)
- Гипотеза: Сглаживание параметров уменьшает вариативность финальной точки.
- Изменение: EMA поверх весов c `alpha ∈ {0.99, 0.995}`; валидировать EMA-снэпшот.
- Датасет: DS1
- Критерий: +0.33%
- Риски: неверный decay может замедлять адаптацию.

## A10 — Тип SPD-нормировки: `cov_type`
- Гипотеза: `trace`-нормировка может быть полезна при нестабильности корреляционной матрицы.
- Изменение: `cov_type ∈ {'corr', 'trace'}`
- Датасет: DS1
- Критерий: +0.33%
- Риски: потеря относительных масштабов корреляций.

## A11 — Subject embeddings: включение/выключение
- Гипотеза: Без эмбеддингов может улучшиться перенос на новые субъекты; с эмбеддингами — выше качество в замкнутом множестве.
- Изменение: `use_subject_embed ∈ {True, False}`
- Датасет: DS1 → DS2/DS3
- Критерий: +0.33%
- Риски: падение на мультисубъектных данных при отключении.

## A13 — Subject-aware CV (оценка)

# Реализация: подсказки по коду

## WeightedRandomSampler (скetch)
```
# после получения train_idx, val_idx
from torch.utils.data import WeightedRandomSampler
labels = np.array([s['label'] for s in samples])
train_labels = labels[train_idx]
class_counts = np.bincount(train_labels, minlength=cfg['model']['n_classes'])
weights_per_class = (len(train_labels) / (len(class_counts) * np.clip(class_counts, 1, None)))
weights = weights_per_class[train_labels]
sampler = WeightedRandomSampler(weights=torch.DoubleTensor(weights), num_samples=len(train_idx), replacement=True)
train_loader = DataLoader(ChiscoSubset(dataset, train_idx), batch_size=cfg['training']['batch_size'], sampler=sampler, shuffle=False, num_workers=cfg['training']['num_workers'], pin_memory=cfg['training']['pin_memory'])
```

## EMA (скetch)
```
# инициализация
ema = {k: v.detach().clone() for k, v in model.state_dict().items()}
alpha = 0.99
# после каждого optimizer.step()
with torch.no_grad():
    for k, v in model.state_dict().items():
        ema[k].mul_(alpha).add_(v, alpha=1 - alpha)
# для валидации (опция):
backup = {k: v.detach().clone() for k, v in model.state_dict().items()}
model.load_state_dict(ema)
val_metrics = evaluate(...)
model.load_state_dict(backup)
```
