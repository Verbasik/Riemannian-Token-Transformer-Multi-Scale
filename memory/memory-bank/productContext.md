# Контекст продукта

## Проблемы
- Шум и артефакты в ЭЭГ, высокая вариативность между субъектами.
- Дисбаланс классов (импакт на macro-F1 и устойчивость обучения).
- Ограниченная обучающая выборка и межсубъектные сдвиги (subject shift).

## Решения (реализованы)
- Риманово представление токенов: SPD-корреляции из окон EEG → logm → векторизация (устойчивый признак).
- Двухмасштабная токенизация + Transformer encoder + attention pooling.
- Гибридная нормализация: subject-wise центрирование + глобальное масштабирование (уменьшает baseline shifts).
- Subject embeddings: адаптация к индивидуальным особенностям (включены по умолчанию).
- Class-Balanced Focal Loss для компенсации дисбаланса.
- Стратифицированные K-Fold сплиты; ранняя остановка и сохранение лучшей модели.

## Альтернативы (в коде)
- Классические ML-бейзлайны на ручных признаках: `Pipeline/feature_engineering.py` + `RandomForest`/`XGBoost` (`Pipeline/train_classical_ml.py`).

## Ценность
- Повышение устойчивости к межсубъектным различиям за счёт гибридной нормализации и персонализации.
- Реплицируемый пайплайн (артефакты и метрики сохраняются), удобный для исследований и абляций.

## Пользовательские сценарии
- Запуск обучения DL-модели: `python3 Pipeline/train.py`.
- Сравнение с классическими ML: `python3 Pipeline/train_classical_ml.py`.
- Анализ результатов: чтение `Train/results/**/metrics.json`, артефактов чекпойнта.

## Границы
- Ожидаются предобработанные pkl (не включён этап сырой предобработки MNE в данном конвейере обучения).
- В текущем состоянии используется первый фолд (при необходимости расширить до всех 5).

## Новые наблюдения (баланс 4 субъектов)
- Сборка идеально сбалансированного датасета по метаклассам из `sub-01..sub-04` (11,464 образцов; 1,433/класс; 51.9% удалено) привела к худшей валидации, чем обучение на одном `sub-04`.
- Интерпретация: сильное undersampling ухудшает усвоение частых паттернов, а межсубъектная гетерогенность повышает сложность задачи; при наличии `ClassBalancedFocalLoss` предпочтительнее не удалять данные, а балансировать весами/семплингом.
- Рекомендация продуктового уровня: честная оценка переносимости через subject-aware CV (StratifiedGroupKFold/LOSO) и отказ от агрессивного undersэмплинга в пользу re-weighting/oversampling менее представленных классов.
