### 8.2.5 Выходные признаки (Features) модели

После обработки согласно пайплайну выше, модель RTTMultiScale
оперирует следующими представлениями данных:

#### Первичные обработанные признаки (Primary Processed Features):

1. **SPD-векторы Римановой геометрии** (64 признака per window)
   - Источник: Log-Euclidean отображение ковариационной матрицы (6.1)
   - Извлечение: OAS shrinkage с параметром oas_min_alpha=0.1
   - Размерность: N_windows × 64, где N_windows зависит от стриданияокна
   - Смысл: Инвариантное к амплитудным артефактам представление
     межканальных корреляций, устойчивое на геометрии SPD-многообразия

2. **Спектральные полосы (вложенные в ковариацию)**
   - Источник: ЭЭГ на частоте 500 Гц (отресемплировано с 1000 Гц)
   - Смысл: Ковариационная матрица неявно кодирует энергию в диапазоне
     0-250 Гц (Nyquist) через меж-канальные зависимости

#### Вторичные (архитектурные) признаки (Secondary/Architectural Features):

3. **Мультимасштабные токены** (8 признаков уровня масштабирования)
   - Малые окна: window_size=128 samples (256 мс), stride=96 (192 мс)
   - Большие окна: window_size=256 samples (512 мс), stride=128 (256 мс)
   - Для каждого окна вычисляется SPD-вектор (признак 1)
   - Ключевая идея: двумасштабная ВАЖНОСТЬ временного контекста

4. **Тематические эмбеддинги** (16 признаков)
   - Размерность: subject_embed_dim=16
   - Источник: категориальная информация о субъекте
   - Смысл: Мягкая персонализация — отдельный эмбеддинг для каждого
     испытуемого, кодирующий индивидуальные особенности ЭЭГ

5. **Позиционные кодирования** (d_model=128 признаков)
   - Источник: синусоидальное позиционное кодирование (Vaswani et al. 2017)
   - Смысл: Порядковая информация о расположении токена в последовательности

#### Финальное представление в классификаторе (Classifier Input Features):

6. **Объединённый вектор признаков** (128+16+CLS=~145 признаков)
   - Объединение attention-pooled представления (128 dims из Transformer)
   + subject embedding (16 dims)
   + CLS-токен (представление всей последовательности)
   - Подаётся в MLP-голову для классификации

#### Статистика признаков по заданиям:

| Уровень обработки | Размерность | Источник | Кол-во каналов (C) |
|---|---|---|---|
| Сырой сигнал (ЭЭГ) | C × T = 125 × 3300 | 500 Гц, 6.6 сек | 125 (124 active) |
| Линейная проекция каналов | 24 × T | proj_channels=24 | 24 |
| Ковариационные матрицы | 24 × 24 per window | OAS в каждом окне | N/A |
| SPD-векторы (Riemannian) | 64 per window | Log-Euclidean vectorization | 64 |
| Мультимасштабные токены | 128 × N_tokens | После проекции SPD | 128 |
| Subject embeddings | 16 per sequence | Learnable | 16 |
| **Финальный вектор классификатора** | **145–170** | Concatenated | **145** |

#### Итоговая статистика:

**Общее количество используемых признаков (feature set):**
- На этапе ковариации: 24 каналов (после proj_channels) × 24 каналов = 576 элементов матрицы
- На этапе SPD-вектора: 64 элемента (верхний треугольник + диагональ ковариационной матрицы)
- На этапе Transformer: 128 + 16 (subject embedding) = 144 скрытых признака
- На выходе MLP-головы: 8 логитов для 8 мета-классов

**Ключевой момент:** Модель использует не фиксированный вектор признаков,
а ДИНАМИЧЕСКИЙ набор токенов переменной длины (зависит от длины сигнала),
где каждый токен кодирует SPD-представление окна и контекст относительно других токенов.

```
### 8.0.5 Архитектурный выбор: стратегия обучения (одна общая модель vs отдельные модели)

#### Постановка выбора:

Для многосубъектного EEG-классификационного пайплайна существуют два основных подхода:

**Вариант A: Subject-Independent (SI) — одна общая модель для всех субъектов**
- Одна общая архитектура RTTMultiScale, обученная на объединённых данных всех субъектов
- Персонализация через subject embeddings (обучаемые векторные представления каждого субъекта)
- Преимущества: обобщение на новых субъектах, эффективное использование данных
- Недостатки: необходимо иметь новые примеры для вычисления subject embedding в inference

**Вариант B: Subject-Dependent (SD) — отдельная модель для каждого субъекта**
- Для каждого из 5 субъектов обучается отдельная модель
- Преимущества: максимальное качество на известных субъектах
- Недостатки: плохая переносимость на новые субъекты, дублирование парамтров

#### Принятое решение в текущем baseline: **Вариант A (Subject-Independent с персонализацией)**

Все модели обучаются с использованием одной архитектуры RTTMultiScale,
которая:

1. **Параметры, общие для всех субъектов:**
   - Слой линейной проекции каналов (C → C')
   - Трансформер-энкодер (weights, биасы, attention patterns)
   - MLP-голова для классификации

2. **Параметры, специфичные для каждого субъекта:**
   - Subject embedding (subject_embed_dim=16): вектор для каждого субъекта,
     конкатенируется с выходом трансформера перед MLP-головой
   - Это позволяет модели учитывать индивидуальные отличия в ЭЭГ-физиологии
     (амплитуда, фазовые сдвиги, шум) БЕЗ обучения отдельных моделей

#### Обоснование выбора варианта A:

| Критерий | Вариант A (SI + embeddings) | Вариант B (SD, отдельные модели) |
|---|---|---|
| Переносимость на новых субъектов | ✅ Высокая | ❌ Невозможна без переобучения |
| Качество на известных субъектах | ✅ Хорошее | ✅✅ Очень хорошее |
| Эффективность параметров | ✅ N × embeds + shared params | ❌ N × full_model (дублирование) |
| Интерпретируемость subject bias | ✅ Видны эмбеддинги | ❌ Скрыты в весах разных моделей |
| Требуемый объём данных на субъект | ✅ Меньше (300 проб) | ❌ Больше (для отдельного обучения) |
| Цель для PhD-исследования | ✅ Воспроизводимый baseline для ALL subjects | ❌ Специализированное решение |

#### Status: почему это выбор именно SI, а не SD:

1. **Текущий этап (Phase 4B)** сфокусирован на построении воспроизводимого baseline,

2. **Следующий этап** (Phase 5) планирует исследование per-subject fine-tuning и
