Консолидируй недавние сессионные заметки в стабильные файлы памяти.

## Процесс

### 1. Собери кандидатов
Прочитай файлы в `memory-bank/.local/SESSIONS/` с неотмеченными пунктами в секции "Знания для промоции".

### 2. Для каждого кандидата — реши

- **ADD**: Новое знание, отсутствующее в стабильных файлах → добавь в канонический файл.
- **UPDATE**: Существующий факт нужно скорректировать → обнови на месте, поставь новую дату.
- **SKIP**: Знание уже существует и актуально → отметь пункт как выполненный.
- **CONFLICT**: Новая информация противоречит стабильному факту → помечай для ревью человеком, НЕ перезаписывай молча.
- **CONSTITUTION_CONFLICT**: Новая информация конфликтует с принципом конституции → НЕ применять, пометить для ревью человека.

### 3. Примени изменения
Для каждого ADD или UPDATE:
- Запиши в канонический файл (по правилам владения из INDEX.md).
- Добавь "Верифицировано: [сегодня]".
- Отметь пункт промоции в сессионном файле.

### 4. Очисти сессии
- Файлы со всеми отмеченными пунктами и старше 14 дней → перенеси в `ARCHIVE/`.
- Файлы старше 30 дней без исключений → перенеси в `ARCHIVE/`.
- Файлы в `ARCHIVE/` старше 90 дней → предложи удаление.

### 5. Отчёт

```
## Консолидация — [дата]

### Промотировано
- [факт] → [файл] (ADD/UPDATE)

### Конфликты (нужен ревью человека)
- [противоречащие факты и их источники]

### Архивировано
- [N] сессионных файлов перемещено

### Осталось
- [N] файлов с необработанными пунктами
```
