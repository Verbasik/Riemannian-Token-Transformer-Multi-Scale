# EEG→Text Pipeline (Phase 4B) — Project Brief

Цель проекта: построить воспроизводимый пайплайн обучения и оценки моделей для распознавания воображаемой речи по ЭЭГ с агрегацией исходных 39 классов в 8 мета‑классов. Поддерживаются две линии: глубокая модель RTT‑MultiScale (риманов токен‑трансформер) и классические ML с признакостроением.

## Цели и результаты
- Основная цель: классификация ЭЭГ в 8 мета‑классов на основе датасета Chisco (задача «imagine»).
- Ключевые результаты:
  - Скрипт обучения глубокой модели: `train.py` (сохранение `metrics.json` и весов `best_model.pt`).
  - Скрипт классического ML baseline: `train_classical_ml.py` (извлечение признаков и сравнение с baseline Phase 4B‑2).
  - Dry‑run тест: `test_dryrun.py` (быстрая проверка корректности пайплайна; требует обновления сигнатур, см. Примечания).

## Требования и ограничения
- Данные: предпроцессинг хранится в `Chisco/derivatives/preprocessed_pkl/`, сопоставления в `Chisco/json/` (`classnumber.json`, `textmaps.json`, `metaclasses.json`).
- Конфигурация: формируется в `config.default_config()`; задаёт пути, нормализацию, параметры модели, тренировку, оптимизатор/шедулер, лосс и каталоги артефактов.
- Среда: Python 3.10+, PyTorch 2.x, NumPy, SciPy, scikit‑learn, PyWavelets; XGBoost опционально.
- Вычислительные ресурсы: GPU желательно (AMP поддерживается автоматически при наличии CUDA), CPU поддерживается.
- Ограничения: файлы данных не входят в репозиторий; каналы для исключения задаются в конфиге (`exclude_channels=[124]`).

## Область и границы
- Включено: загрузка и стратифицированное разбиение данных, нормализация (в т.ч. гибридная subject‑wise+global), обучение, оценка, сохранение артефактов; baseline с признаками (PSD/соотношения/статистики/вейвлет/Hjorth).
- Исключено: онлайновая/реального времени инференс‑служба; полноценный текстовый декодер; автоматический менеджмент данных.

## Ключевые метрики
- Классификация: `accuracy`, `f1_macro`, `precision_macro`, `recall_macro`, `balanced_accuracy`.
- Валидация: первый фолд K‑fold (по умолчанию 5), метрики пишутся в `Train/results/.../metrics.json`.

## Приёмочные критерии
- Запуск `python train.py` завершается без ошибок, сохраняет метрики и лучшую модель.
- Запуск `python train_classical_ml.py` завершается, сохраняет метрики baseline и сравнение с Phase 4B‑2 (числа из кода baseline: 27.72% acc / 25.69% F1‑macro для справки).
- Конфигурация и пути корректируются одной точкой (`config.default_config`).

## Риски и предположения
- Риск отсутствия данных по указанным путям (`Chisco/...`). Требуется локальная подготовка данных.
- Возможная несогласованность тестового скрипта `test_dryrun.py` с текущими сигнатурами билдеров (возвращаемые значения функций изменились).
- Объём данных и дисбаланс классов — используется Class‑Balanced Focal Loss для смягчения.

## Примечания по запуску
- Глубокая модель: `python train.py`
- Классический ML baseline: `python train_classical_ml.py`
- Dry‑run (быстрая валидация пайплайна): `python test_dryrun.py` (см. замечание о сигнатурах)
