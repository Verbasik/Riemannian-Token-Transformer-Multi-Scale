Проанализируй память проекта и выяви пробелы.

## Процесс

### 1. Загрузи контекст
Прочитай `memory-bank/INDEX.md`, `memory-bank/CONSTITUTION.md`, `memory-bank/ARCHITECTURE.md`, `memory-bank/DECISIONS.md`.

### 2. Проверь полноту архитектуры
Для каждого модуля в ARCHITECTURE.md:
- Есть ли соответствующий файл в AREAS/?
- Описаны ли взаимодействия между модулями?
- Есть ли пробелы в описании потоков данных?
Пометь пробелы как вопросы.

### 3. Проверь согласованность решений
Для каждого ✅ Active решения в DECISIONS.md:
- Согласуется ли с ARCHITECTURE.md?
- Согласуется ли с CONVENTIONS.md?
- Согласуется ли с CONSTITUTION.md?
Пометь расхождения как вопросы.

### 4. Проверь покрытие подсистем
- Все ли значимые директории кодовой базы имеют файл в AREAS/?
- Все ли паттерны, упомянутые в AREAS/, существуют в PATTERNS/?

### 5. Проверь актуальность
- Есть ли файлы с датой верификации старше 60 дней?
- Есть ли OPEN_QUESTIONS.md с нерешёнными вопросами старше 30 дней?

### 6. Запиши результат
Запиши обнаруженные вопросы в `memory-bank/OPEN_QUESTIONS.md`.
Формат каждого вопроса:

| # | Вопрос | Контекст | Поднят | Владелец | Статус |
|---|--------|----------|--------|----------|--------|
| N | [конкретный вопрос] | Обнаружен memory-clarify: [причина] | [сегодня] | — | Открыт |

### 7. Отчёт

```
## Результат memory-clarify — [дата]

### Обнаружено пробелов: [N]

### По категориям:
- Архитектура: [N] пробелов
- Решения: [N] расхождений
- Подсистемы: [N] непокрытых
- Устаревание: [N] файлов

### Добавлено в OPEN_QUESTIONS.md: [N] вопросов

### Рекомендации
1. [конкретное действие]
```
