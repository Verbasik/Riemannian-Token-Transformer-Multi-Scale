# CONSTITUTION

Ненарушимые принципы проекта. Высший приоритет при конфликтах с любыми другими файлами memory bank.
Все skills, решения и изменения архитектуры ОБЯЗАНЫ проверяться на соответствие этим принципам.

## Contract

- when: любое архитектурное решение; выбор технологий; ревью кода; промоция знаний
- prereq: агент собирается принять решение или изменить canonical-файл
- reads: только этот файл
- writes: никогда — только предложения для ревью человека
- success: решение проверено на соответствие; конфликт явно обозначен если найден
- on_fail: если решение конфликтует с принципом -> остановить промоцию; пометить `CONSTITUTION_CONFLICT`; вынести на review

## Failure routes

- `CONSTITUTION_CONFLICT` -> остановить промоцию -> зафиксировать конфликт в `CURRENT.md` -> вынести на human review
- Нарушение допустимо только с явным обоснованием и одобрением человека -> зафиксировать в `ADR/`

## Review rules

- Конституция имеет приоритет над всеми остальными файлами memory bank.
- Изменения требуют: документации причины + обоснования + одобрения человека + ADR.
- Агент НИКОГДА не перезаписывает этот файл автоматически — только предлагает изменения.
- Все решения в `DECISIONS.md` и `ADR/` должны проверяться на соответствие этим принципам.
